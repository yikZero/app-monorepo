# OneKey Platform Patterns

Patterns specific to this codebase and its target platforms. These come from real Sentry crashes and production incidents.

## 1. Android Crash Patterns

### MIUI / HyperOS Splash Screen

### Layout Re-measurement
`tabBarHidden` changes need layout update â€” use `LayoutAnimation` or `forceUpdate`.

### OOM with Images
Check bitmap memory limits. Use `resizeMode` and memory-efficient loading. Watch for large images loaded without dimensions.

## 2. iOS Crash Patterns

### EXC_BAD_ACCESS with nil strings
Always check for nil before NSString operations in TurboModules. If a JS value can be `undefined`/`null`, the native side must handle it.

### expo-image crashes
Validate URI before passing to Image component:
```typescript
if (!uri || uri.length === 0) return null;
```

### Stack overflow in recursive calls
Debounce recursive operations and add depth limits.

## 4. BigNumber / Numeric Type Safety

### decimals might be string from API
```typescript
// Bug: decimals could be "18" (string) from JSON
new BigNumber(value).shiftedBy(-decimals);

// Fix: ensure number type
new BigNumber(value).shiftedBy(-Number(decimals));
```

### Division by zero
```typescript
// Always guard
const rate = total > 0 ? amount / total : 0;
```

### String vs Number in API responses
API responses may return `"100"` or `100` â€” normalize type before arithmetic.

## 5. Cross-Platform Pitfalls

### Extension (MV3)
- Service worker lifecycle: don't rely on long-lived state
- CSP: no eval / dynamic script injection
- Permissions: least-privilege, validate host permissions

### Mobile (React Native)
- Background/foreground transitions: async storage consistency
- Secure storage for secrets (not AsyncStorage)
- WebView: origin isolation, message bridge validation

### Desktop (Electron)
- IPC validation: don't expose privileged APIs to renderer
- `nodeIntegration` off unless justified, `contextIsolation` on
- File system access must be scoped

### Web
- XSS, CSP, CORS, storage leakage
- Bundle/runtime differences vs RN/Electron

## 6. Build Failure Risks

### extraResources referencing generated files
If electron-builder config references a file generated by a script, and the script can skip execution (e.g., platform-specific tools), the build fails silently.

**Fix**: Create placeholder files when skipping generation.

### Scripts with early exit without cleanup
```bash
# Bug: exits without creating expected output
if ! command -v tool &> /dev/null; then
    echo "Tool not found"
    exit 0  # Build expects output file!
fi

# Fix: create placeholder before exit
if ! command -v tool &> /dev/null; then
    mkdir -p "$OUTPUT_PATH"
    touch "$OUTPUT_PATH/expected-file"
    exit 0
fi
```

## 7. Accidental File Commits

Check the diff for files that shouldn't be in the repo:
- `.DS_Store`, `Thumbs.db`
- `.env.local`, `.env.production`
- `node_modules/` anything
- `*.log` files
- Binary files that belong in CDN

## 8. CI Workflow Gaps

- File generation steps should have verification steps after them
- Platform-specific steps should handle cross-platform CI
- Critical paths should fail fast with clear errors, not succeed silently

## 9. Script Accuracy

Check shell scripts for:
- Claims like "only available on macOS" when the tool exists on Linux too
- Incorrect version requirements
- Misleading prerequisite descriptions
- Error messages that don't match what actually went wrong
